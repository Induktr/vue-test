# Руководство по Тестированию "Insight Keeper v2.0"

### 1. Введение: Тестирование как Подтверждение Усвоенных Навыков

Этот документ описывает **систематический подход (Принцип 9)** к тестированию. Тестирование здесь — это **подтверждение того, что вы усвоили навыки (Принцип 1)** и ваш код работает корректно.

### 2. Стратегия Тестирования

*   **Фокус:** Модульное тестирование основных функций `composables` и интеграционное тестирование UI-компонентов.
*   **Инструменты:** Jest (для логики), Vue Test Utils (для компонентов).

### 3. Тестовые Сценарии (TC)

#### **TC-1: Система Тегирования**
*   **TC-1.1:** Проверить `addTags()`: добавление новых уникальных тегов, игнорирование дубликатов.
*   **TC-1.2:** Проверить `allUniqueTags`: корректное агрегирование всех уникальных тегов из нескольких постов.

#### **TC-2: Система Фильтрации**
*   **TC-2.1:** Проверить `filteredPosts`: фильтрацию по `searchQuery`.
*   **TC-2.2:** Проверить `filteredPosts`: фильтрацию по выбранному тегу.
*   **TC-2.3:** Проверить `filteredPosts`: фильтрацию по обоим параметрам одновременно.

#### **TC-3: Система Аналитики Постов**
*   **TC-3.1:** Проверить `analyzePost()`: корректный анализ количества свойств и их типов.

#### **TC-4: Система Пакетных Операций**
*   **TC-4.1:** Проверить `archiveOldPosts()`: удаление старых постов, сохранение нужного количества.

#### **TC-5: Система Приоритетов**
*   **TC-5.1:** Проверить `setPriority()`: успешное обновление приоритета с валидным числом.
*   **TC-5.2:** Проверить `setPriority()`: игнорирование невалидного приоритета (`NaN`, не целое число).
*   **TC-5.3:** Проверить `postColor`: корректное возвращение цвета в зависимости от приоритета.

### 4. Критерии Успеха

*   Все тесты проходят.
*   Вы четко понимаете, почему каждый тест написан именно так, и какой навык он подтверждает.
