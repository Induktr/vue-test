# "Insight Keeper v2.0" FAQ: Обзор Тренировки

### 1. Введение: Вопросы для Роста

Это FAQ — ваш быстрый гид по проекту. Здесь мы отвечаем на вопросы, которые помогут вам понять **цель и философию (Принцип 1)** "Insight Keeper v2.0" как инструмента обучения.

### 2. Общие Вопросы о Проекте

**Для чего нужен этот проект?**
Это **"тренажер"** для отработки и глубокого усвоения конкретных методов JavaScript, TypeScript и Vue.js, указанных в Roadmap. Главная ценность — **практическое применение (Принцип 1)** знаний.

**Какие "инсайты" я получу?**
Вы научитесь эффективно работать с **массивами и Set (Принцип `...spread`, `...rest`)**, создавать **продвинутые фильтры (`filter`, `map`)**, анализировать структуры данных (`Object.keys`, `typeof`), управлять состоянием Vue.js в сложных сценариях и поймете различия между Composition API и Options API на практике.

### 3. Технические Вопросы

**Почему Vue.js?**
Vue.js выбран за его **гибкость** и возможность практиковать как **Composition API**, так и **Options API**, что является частью нашей учебной задачи. Это позволяет на практике сравнить их эффективность.

**Есть ли бэкенд?**
Нет. Этот проект **сознательно** сфокусирован на **Front-end логике и реактивности (Принцип 12 - прагматизм)**. Все данные хранятся в клиентском состоянии для упрощения.

### 4. Вопросы по Тренировке

**Каждый этап — это отдельный навык?**
Да. Каждый "Этап" и "Задача" в Roadmap сфокусированы на отработке конкретного навыка или метода JS/TS/Vue.js. Это **системный подход (Принцип 9)** к обучению.

**Зачем повторять на Options API?**
Это **критический "челлендж"**. Цель — не просто "сделать", а **почувствовать дискомфорт и преимущества** Composition API на контрасте, что является самым ценным **инсайтом (Принцип 1)**.
