# Руководство по Оптимизации "Insight Keeper v2.0"

### 1. Введение: Оптимизация как Часть Усвоения Навыков

Оптимизация здесь — это не про миллионы пользователей. Это про **понимание, как Vue.js и JavaScript работают под капотом**, чтобы ваш код был **эффективным и здоровым (Принцип 5)**.

### 2. Цели Оптимизации (для MVP)

*   **NFR-1 (Плавность UI):** Приложение должно оставаться полностью отзывчивым при наличии до 500 постов.
*   **NFR-2 (Эффективность):** Использование `computed`-свойств для фильтрации не должно вызывать видимых задержек.

### 3. Техники Оптимизации

*   **Vue Reactivity System:**
    *   Используйте `ref` и `reactive` осознанно.
    *   Применяйте `computed` для всех производных состояний (фильтрация, агрегация тегов), чтобы Vue сам управлял их кэшированием.
*   **JavaScript Методы:**
    *   Предпочитайте `map`, `filter`, `reduce` вместо ручных циклов, где это уместно. Они часто более читаемы и могут быть оптимизированы движком JS.
*   **Инструменты:**
    *   **Vue DevTools:** Используйте Profiler для выявления компонентов, которые перерисовываются слишком часто.
    *   **Performance Tab (Chrome DevTools):** Для анализа производительности и плавности UI.
