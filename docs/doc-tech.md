# Техническая Документация "Insight Keeper v2.0"

### 1. Введение: Технический План Вашего Развития

Этот документ — ваш **технический план (Принцип 8)**. Он описывает структуру, инструменты и методы, которые вы будете использовать для достижения **целей обучения (Принцип 1)**.

### 2. Технологический Стек

*   **Фреймворк:** Vue.js 3 (Composition API и Options API для сравнения)
*   **Язык:** TypeScript
*   **Сборщик:** Vite
*   **Управление состоянием:** Встроенная система реактивности Vue.js.

### 3. Архитектура: "Учебный Полигон"

Проект следует структуре Vue.js, с акцентом на разделение логики и представления.

insight-keeper-v2/
├── src/
│ ├── assets/ # Статические ресурсы
│ ├── components/ # Переиспользуемые UI-компоненты
│ ├── composables/ # Vue Composition API функции (ваша основная логика)
│ ├── types/ # TypeScript типы
│ ├── App.vue # Главный компонент
│ └── main.ts # Точка входа
└── public/

```typescript
// src/types/post.ts
export interface Post {
  id: string;
  title: string;
  content: string;
  tags: Set<string>; // Для демонстрации Set
  priority: number; // От 1 до 3
  type: 'Insight' | 'Task' | 'Quote';
  createdAt: Date;
}

5. Ключевые Функции (Реализация)

    Управление Постами (Composition API):

        Все функции для работы с коллекцией постов будут реализованы в src/composables/usePosts.ts.

        Используйте ref и reactive для состояния, computed для производных данных.

    Генерация ID: Используйте cuid или простой счетчик для генерации уникальных ID постов.

6. Vue Composition API vs. Options API

    Цель: После реализации всего функционала на Composition API, ключевая учебная задача — попробовать воссоздать часть или весь функционал на Options API.

    Фокус: Наблюдайте за тем, где Composition API делает код более читаемым, модульным и гибким, а Options API становится громоздким (например, при управлении несколькими связанными состояниями).

# Документация Дизайна "Insight Keeper v2.0"

### 1. Философия: Дизайн как Инструмент Фокуса

Наш дизайн — **минималистичен и функционален**. Он служит для **устранения отвлечений** и помогает сосредоточиться на **содержании (инсайтах)**.

### 2. Цветовая Палитра

*   **Фон:** Светло-серый (`#F5F5F5`) — нейтральность, чистота.
*   **Карточка Поста:** Белый (`#FFFFFF`) — акцент на контенте.
*   **Акцентные Элементы (кнопки, теги):** Мягкий синий (`#3498DB`) — спокойствие, ясность.
*   **Цвета Приоритетов:**
    *   Красный (`#E74C3C`): Высокий.
    *   Желтый (`#F1C40F`): Средний.
    *   Зеленый (`#2ECC71`): Низкий.

### 3. Типографика

*   **Шрифт:** Sans-serif (например, `Roboto` или `Inter`) — современный, чистый, легко читаемый.
*   **Иерархия:** Четкая иерархия размеров для заголовков, основного текста и тегов.

### 4. Структура Интерфейса

*   **Главный Экран:**
    *   Строка поиска (по названию) сверху.
    *   Кнопки фильтрации по тегам (`allUniqueTags`) в виде горизонтального списка.
    *   Список карточек постов.
    *   Кнопки для "пакетных операций" (например, "Архивировать старые посты").
*   **Карточка Поста:**
    *   Заголовок, краткое содержимое.
    *   Теги (небольшие, скругленные кнопки).
    *   Индикатор приоритета (цветовой кружок/полоска).
    *   Дата создания.
    *   Кнопки действий (редактировать, удалить).
