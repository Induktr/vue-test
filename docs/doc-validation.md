# Правила Валидации "Insight Keeper v2.0"

### 1. Введение: Валидация как Гарант Целостности Данных

Валидация — это **первая линия защиты (Принцип 5)** ваших данных. Она обеспечивает **целостность (Принцип 7)** системы.

### 2. Правила Валидации

#### **Поле: `title` (для нового поста)**
*   **Правило 1:** Обязательное поле.
    *   *Сообщение:* "Заголовок поста обязателен".
*   **Правило 2:** Минимальная длина — 3 символа.
    *   *Сообщение:* "Заголовок должен быть длиннее 3 символов".
*   **Правило 3:** Максимальная длина — 100 символов.
    *   *Сообщение:* "Заголовок должен быть короче 100 символов".

#### **Поле: `priority` (при установке приоритета)**
*   **Правило 1:** Должно быть числом.
    *   *Сообщение:* "Приоритет должен быть числом".
*   **Правило 2:** Должно быть целым числом.
    *   *Сообщение:* "Приоритет должен быть целым числом".
*   **Правило 3:** Должно быть в диапазоне от 1 до 3.
    *   *Сообщение:* "Приоритет может быть только 1, 2 или 3".

### 3. Техническая Реализация

*   **Frontend (Vue.js):** Валидация будет реализована с использованием реактивной логики Vue (например, `computed`-свойства для ошибок).
